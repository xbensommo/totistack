# Totistack v2

Modular scaffolding for building real business systems faster.

Totistack v2 exists for one reason: rebuilding the same business software foundation over and over is a waste of time.

This toolkit gives you a cleaner starting point for serious systems such as CRMs, booking platforms, dashboards, internal tools, CMS platforms, and client portals.

## What Totistack gives you

- a global CLI
- structured project generation
- installable apps and features
- preset-driven scaffolding
- collection-aware generation
- centralized project config and branding
- runtime registration for routes, stores, services, and providers
- Firestore support through `@xbensommo/shard-provider` when Firestore is used

## Core command

```bash
toti create <project-name>
```

Example:

```bash
toti create my-business-app
```

## Why this exists

Most business projects repeat the same patterns:

- authentication
- RBAC
- CRUD flows
- dashboards
- collections
- admin pages
- routing
- project config
- branding
- integrations

Totistack cuts that repeated setup work and gives you a stronger structure from day one.

## Who this is for

### Developers

If you are tired of rebuilding the same foundation on every project, this is for you.

Totistack helps you move faster without turning the codebase into a mess later.

### Teams

If you need consistency across multiple projects, apps, and feature modules, Totistack gives you one way of doing things properly.

### Business owners

You do not need to know the internal architecture to benefit from this. The value is simple: faster project setup, cleaner delivery, easier long-term expansion.

## What matters most

Totistack is not built for toy examples.

It is built for:

- real business use
- modular growth
- maintainable projects
- cleaner extension paths
- serious internal and client systems
